# Example: a worked artifact produced by High Council

This folder is an **end-to-end demonstration of what the `high-council` skill produces** —
not a set of hand-written docs. Every file here was generated by running the skill in three
phases over a single problem, with the models drafting, disagreeing, revising, and judging
each other's work.

## The task

Design the operator kit for a startup's "Phase 0" — a 90-day, ~$6k market-validation test
for an independent reliability-standard business (evaluating how often AI coding agents
silently report work as "done" when it isn't). The kit had to be concrete enough to run next
week, with checkable acceptance criteria and hard business rules preserved verbatim.

## How it was produced (three council passes)

1. **Orchestrate** (`high-council` mode a) — three models each proposed a build plan; the
   router found they split the work differently, triggered one revision round, and the judge
   synthesized [`00-build-plan.md`](00-build-plan.md): a two-workstream decomposition with
   per-artifact acceptance criteria.
2. **Execute** — the two workstreams were built by two different models in parallel (one
   handled the writing-heavy demand-discovery instruments, the other the spec/formula-heavy
   evaluation machinery), producing
   [`01-workstream-a-demand-discovery.md`](01-workstream-a-demand-discovery.md) and
   [`02-workstream-b-evaluation-integrity.md`](02-workstream-b-evaluation-integrity.md).
3. **Review** (`high-council` mode a) — three models independently reviewed the built kit;
   the router surfaced nine finding-level disagreements, the revision round made each
   reviewer re-verify against the kit text, and the judge adjudicated every one by direct
   recompute/character-check, yielding [`03-review.md`](03-review.md) (a
   `READY-WITH-FIXES` verdict with prioritized, kit-grounded fixes).

The kit files here have had the review's **blocking** and **should-fix** items applied.

## Why this is a good demonstration

- The review caught **real** defects a single model would likely have missed or waved
  through — e.g. a metric whose denominator silently mis-handled non-adjudicable cases, and
  a hard-rule clause that had been paraphrased in buyer-facing copy (dropping "trained on").
- It also **killed false findings** — one reviewer invented a rule the spec never contained;
  another over-graded the kit `NOT-READY`; the judge overruled both by evidence.
- The pattern on show is the one the skill is meant for: **council to decide → direct
  execution to build → council to review.** The council does the judgment; the building
  happens outside it.
